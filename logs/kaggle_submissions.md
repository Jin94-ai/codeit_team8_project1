## 제출 규칙

### 1. 제출 전 필수 사항
- [ ] 팀 Discord/Slack에 제출 예정 공지
- [ ] 실험 로그 작성 완료 (`logs/experiments/exp_XXX.md`)
- [ ] 팀원들 의견 수렴 (필요시)

### 2. 제출 프로세스
1. 로컬에서 모델 검증
2. 제출 파일 생성 (`scripts/make_submission.py`)
3. Kaggle에 제출
4. 이 파일 업데이트 (위 테이블에 추가)
5. 팀원들에게(Discord 등) 결과 공유

### 3. 제출 전략
- **하루 최대 제출**: 2회 권장
- **제출 소진 주의**: 5명 × 2회 = 10회 빠르게 소진 가능
- **낮은 성능 모델도 제출 고려**:
  - 내부 검증 성능이 낮아도 일반화가 잘 되었을 수 있음
  - Public/Private 리더보드 차이 확인 목적

### 4. 제출 후 작업
- [ ] Public Score를 위 테이블에 기록
- [ ] 실험 로그에 Kaggle Score 추가
- [ ] README.md 실험 결과 테이블 업데이트
- [ ] 팀원들과 결과 논의
